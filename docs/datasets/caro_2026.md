## Cómo consultar

```python
from agentes.datos.consultor_bigquery import ConsultorBigQuery

consultor = ConsultorBigQuery()
resultado = consultor.obtener_snow_depth_caro(
    estacion="La Parva",
    fecha_inicio="2023-06-01",
    fecha_fin="2023-09-30",
    qc_status="clean",
    apply_qc=False,  # True para aplicar pipeline QC sobre datos raw
)
```

Consulta SQL directa:

```sql
SELECT station_name, observation_date, snow_depth_cm
FROM `climas-chileno.clima.snow_depth_caro_2026`
WHERE qc_status = 'clean'
  AND basin = 'Maipo'
  AND observation_date BETWEEN '2023-06-01' AND '2023-09-30'
ORDER BY station_name, observation_date;
```

## Hallazgo crítico: no-linealidad SD–elevación

Caro et al. (2026) documentan que en la cuenca del Maipo la acumulación máxima
de nieve ocurre en ~3300 m, con disminución progresiva sobre los 4000 m por
sublimación inducida por viento. Este hallazgo implica:

1. El rango de máxima SD (3000–3500 m) coincide con pendientes críticas EAWS (30–45°).
2. Los modelos transferidos desde Swiss SLF sobreestiman SD en alta elevación y
   subestiman a elevación media — recalibración necesaria en subagente S1.
3. El subagente S1 (PINNs) debe incorporar variables de exposición eólica además
   de elevación y pendiente.

## Limitaciones conocidas

- El formato exacto de los archivos Zenodo debe verificarse con `explorar_caro_2026.py`
  antes de ingestar (puede ser CSV, NetCDF o HDF5).
- El mapeo de columnas en `ingestor_caro_2026.py` cubre los nombres más probables;
  si el dataset usa nombres distintos, actualizar `_normalizar_columnas()`.
- El dataset cubre 2010–2024 con gaps según estación y fuente.
- La columna `sensor_model` puede ser null en estaciones de fuentes con metadata incompleta.
